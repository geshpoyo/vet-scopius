# 言語別：ドット記法と階層の意味

## 概要

このノートは、各プログラミング言語における「ドット（.）」の意味と、階層構造の表現方法を比較・整理したものです。特に Python におけるドット記法が他言語と異なる点を明確にします。

---

## 比較表

|言語|階層の書き方|ドットの意味|
|---|---|---|
|Python|[`services.data.loader`](https://services.data.loader)|フォルダ階層（インポート時のモジュール指定）|
|Java|[`com.example.app`](https://com.example.app)|パッケージ名（フォルダとは別）|
|C#|`MyApp.Services`|名前空間（Namespace）|
|JS / TS|`./services/data/`[`loader.js`](https://loader.js)|拡張子のみ|
|Rust|`services::data::loader`|モジュール階層（`mod`構造）|
|PHP|`App\Services\Data`|名前空間（Namespace）|

---

## 補足解説

### Python

- ドットはフォルダ階層を表す。
    
- `from` [`services.data`](https://services.data) `import loader` のように、フォルダ構造をそのままドットで表現。
    
- `.py` 拡張子は書かない。
    
- フォルダ名にドットを使うと階層と誤認されるため避ける。
    

### Java

- ドットはパッケージ名の区切り。
    
- 実際のフォルダ構造と一致させるのが慣習だが、厳密には別物。
    

### C#

- ドットは名前空間の区切り。
    
- フォルダ構造とは無関係。
    

### JavaScript / TypeScript

- ドットは拡張子（`.js`, `.ts`）にのみ使う。
    
- 階層は `/` で表現。
    

### Rust

- モジュール階層は `::` で表現。
    
- ファイルやフォルダ構造と連動するが、ドットは使わない。
    

### PHP

- 名前空間は `\` で区切る。
    
- フォルダ構造と一致させるのが一般的。
    

---

## まとめ

- ドットで階層を表すのは **Pythonだけ**。
    
- 他言語では、ドットは名前空間やパッケージ名の区切りとして使われる。
    
- Pythonでは、インポート時にフォルダ構造をドットで表現するが、他言語では構文や慣習が異なる。
